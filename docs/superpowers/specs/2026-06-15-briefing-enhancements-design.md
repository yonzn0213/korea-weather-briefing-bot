# 브리핑 기능 추가 설계

- 상태: 승인 대기
- 날짜: 2026-06-15
- 대상 파일: 주로 `worker/src/briefing.ts` (+ 테스트)

## 배경

`@korea_weather_briefing_bot`는 전국 아침 브리핑 봇이다. 현재 브리핑은 비 소식 / 최저·최고기온 / 하늘 / 미세먼지로 구성된다. 사용자가 다음을 원한다.

1. 시간대별 온도
2. 기온에 따른 옷차림
3. 행운의 색깔
4. 최저기온이 "한 번도 표시된 적 없음" → 진단/해결
5. 프로젝트 이름 정리 (서울 → 전국)

모든 기능은 **추가 외부 호출 없이** 구현된다. 시간별 기온(TMP)·최저(TMN)·최고(TMX)는 이미 호출 중인 기상청 단기예보 응답에 함께 들어있고, 옷차림·행운의 색은 순수 함수다. subrequest 무료 한도(45)에 영향 없음.

## 요구사항 및 결정

### 1. 시간대별 온도
- 기상청 응답의 `TMP`(1시간 기온) 카테고리를 파싱한다. (현재는 버려지고 있음)
- 표시 시간: **6·9·12·15·18·21시** 6개.
- 출력 예: `⏰ 6시 18° · 9시 21° · 12시 25° · 15시 26° · 18시 23° · 21시 20°`
- 해당 시간 값이 없으면 그 항목은 생략(나머지만 표시). 전부 없으면 줄 자체 생략.

### 2. 기온별 옷차림 (최저~최고 범위)
표준 "기온별 옷차림" 표를 구간 매핑으로 둔다.

| 기온 | 대표 옷차림 |
|------|------|
| 28°C 이상 | 민소매·반팔·반바지 |
| 23~27°C | 반팔·얇은 셔츠·면바지 |
| 20~22°C | 긴팔·얇은 가디건·면바지 |
| 17~19°C | 맨투맨·얇은 니트·가디건 |
| 12~16°C | 자켓·가디건·야상·청바지 |
| 9~11°C | 트렌치코트·점퍼·니트 |
| 5~8°C | 코트·히트텍·가죽자켓 |
| 4°C 이하 | 패딩·두꺼운 코트·목도리 |

- 최저기온의 옷차림 ~ 최고기온의 옷차림을 범위로 표시.
- 출력 예: `👕 옷차림: 자켓·가디건 ~ 긴팔·얇은 가디건`
- 최저·최고가 같은 구간이면 하나만 표시: `👕 옷차림: 반팔·얇은 셔츠·면바지`
- 최저/최고가 둘 다 없으면 줄 생략.

### 3. 행운의 색깔
- 색 목록(약 12~14개, 이모지 포함)에서 랜덤 1개.
- 출력 예: `🎨 오늘의 행운 색: 청록색 🩵`
- **테스트 가능성**: `buildMessage`는 현재 순수 함수다. 랜덤을 내부에서 호출하지 않고 인자로 주입한다. 예: `pickLuckyColor(rand: () => number)` 형태로 분리하고, `buildMessage`는 이미 고른 색 문자열을 인자로 받거나 `rand` 기본값을 받는다. 구체 시그니처는 구현 계획에서 확정.
- 색 후보(초안): 빨강❤️, 주황🧡, 노랑💛, 초록💚, 파랑💙, 보라💜, 분홍🩷, 청록🩵, 남색, 하늘색, 금색✨, 은색, 연두색, 자홍색.

### 4. 최저/최고기온 fallback (핵심 안정화) — 원인 실측 확정
- **원인 확정(2026-06-15 실측)**: cron이 도는 KST 07:00에 쓰는 `base_time=0500` 발표분 응답 카테고리 = `PCP, POP, PTY, SKY, TMP, UUU, VEC, VVV, WAV, WSD`. **`TMN`·`TMX`가 아예 없음.** 이름 변경이 아니라 0500 발표분이 일최저/일최고를 제공하지 않음. → 최저뿐 아니라 최고도 사실상 못 받고 있었음.
- 해결: 시간별 기온(TMP)을 이미 파싱하므로 이를 fallback으로 사용.
  - 최저기온 = `TMN` 있으면 그 값, 없으면 **오늘 TMP 최솟값**
  - 최고기온 = `TMX` 있으면 그 값, 없으면 **오늘 TMP 최댓값**
- `TMN`/`TMX`가 있는 발표분이면 정확값 우선 사용. 현재 0500 발표에선 항상 fallback이 작동.
- 검증 예(2026-06-15, 강남 nx61/ny125): TMP min 19°C / max 32°C → 최저 19 / 최고 32.
- 한계: 07시 기준 TMP는 06시부터라 새벽 실제 최저와 약간 차이날 수 있음(허용 범위).

### 5. 프로젝트 이름
- 코드/문서는 이미 `korea-weather-bot` / "전국 아침 브리핑 봇"으로 정리되어 있음.
- 남은 것은 **GitHub 저장소명** `seoul-morning-bot` 하나 → 사용자가 GitHub Settings에서 직접 변경 (추천: `korea-weather-briefing-bot`). 코드 변경 없음.

## 영향 범위 / 모듈

- `worker/src/briefing.ts`
  - `Weather` 인터페이스에 `hourly: Record<string, number>`(또는 시간→기온 맵) 추가.
  - `fetchWeather`: `TMP` 파싱 추가, 최저/최고 fallback 계산.
  - 신규 순수 함수: `clothingFor(temp)`, `clothingRange(tmn, tmx)`, `pickLuckyColor(rand)`, `formatHourly(hourly)`.
  - `buildMessage`: 시간대별·옷차림·행운의 색 줄 추가, 행운의 색 인자 주입.
- 테스트(`worker/test/briefing.test.ts`)
  - TMP 파싱, fallback 최저/최고, 옷차림 구간/범위, 행운의 색(주입된 rand로 결정적), 시간대별 포맷, 누락값 생략.
